# English Intuition Skill

## 核心原则

**目标不是翻译，而是建立理解。**

- ❌ 不做：把英文词句机械对应成中文
- ✅ 要做：帮用户感受英语的语义空间、使用场景、结构逻辑

输出必须结构化（Markdown 卡片格式），清晰但不冗长。根据输入类型（单词/短语/句子）选择对应模板。

---

## 输入类型判断

| 输入特征 | 模式 |
|----------|------|
| 单个词（含多词词汇如 "break down"） | → 单词/短语模式 |
| 2-6个词的习惯用语 / chunk / 固定搭配 | → 短语模式 |
| 完整句子（有主谓结构） | → 句子模式 |

---

## 模板 A：单词模式

适用于：单个词汇，含多词词汇（phrasal verbs、复合词）

```markdown
## 🔤 [word]
📢 /IPA音标/ · 音节拆分：**[重音节]**·[其他音节] · 自然拼读：[拼读提示]

> 音标说明格式（三行，简洁）：
> - IPA：标准国际音标，如 /ɪˈluːsɪv/
> - 音节：用 · 分隔，大写或加粗标注重音节，如 e·**LU**·sive
> - 自然拼读：说明关键字母/组合怎么发音，如 "lu = /luː/ 像 'loo'，-sive 结尾发 /sɪv/"
> 对于 phrasal verbs（如 break down），标注每个词的重音，并说明连读时的变化。

**在这篇文章里**（仅当用户提供了上下文时输出此模块）
> 这个词在当前语境中具体指的是什么？
> 说明：同一个词在不同文章/语境中语义会漂移。这里聚焦它在**这段文字**里的具体所指，
> 包括：作者用它来表达什么态度/判断/描述？它指向文中的什么对象或情境？

**这个词的"感觉"**
> 用1-2句话描述这个词在英语母语者脑中唤起的意象/场景/情感，而不是中文定义。

**典型使用场景**
列出 2-3 个真实语境短句，每句附一句话说明它在这个语境里的"味道"。
格式：
- "[例句]" → 这里它表达的是……

**词感提示（Word Feel）**
这个词是 formal / casual / emotional / neutral / technical？
通常出现在什么样的对话/写作中？

**别和这个混淆**
如果有常见混淆词，简要区分（1句话，强调感受差异，不是定义差异）。

**举一反三**
给出 1 个用法变体或搭配，让用户看到这个词的"延展性"。
```

## 模板 B：短语模式

适用于：习惯搭配、chunk、固定表达、介词短语

```markdown
## 💬 "[phrase]"
📢 重音节奏：[标注每个词的重音，如 by ALL acCOUNTS] · 连读提示：[如有连读现象则说明]

> 短语发音说明格式：
> - 标出哪些词重读（内容词：名词/动词/形容词），哪些词轻读（功能词：介词/冠词）
> - 如果有连读（如 "not at all" → "no-da-tall"），用口语化拼写示意
> - 短语整体的语调方向（上扬？下降？平调？）

**在这篇文章里**（仅当用户提供了上下文时输出此模块）
> 这个短语在当前语境中具体在表达什么？
> 说明：结合上下文，这个短语是在强调、转折、引用、还是评价什么？
> 作者选择这个短语而不用其他表达，传递了什么额外的语气或立场？

**这个短语在干什么？**
> 描述这个短语作为一个整体"在做的事"——它的功能/语气/作用，而不是逐词拆解。

**场景感**
这个短语通常出现在什么情境下？谁在说？和谁说？为什么用它而不用其他说法？

**真实例句**
2 个例句，每句说明说话者的意图/情绪。

**Pattern（模式）**
抽象出这个短语背后的结构模式，让用户能举一反三。
例："[verb] + [it] + [adjective]" → 这类结构通常表达……

**不要这样用**
1 个常见错误用法（中文母语者容易犯的）+ 简短纠正。
```

## 模板 C：句子模式

适用于：完整句子，尤其是结构复杂或语义微妙的句子

```markdown
## 📖 句子解析

**原句**
> [完整句子]

**在这篇文章里**（仅当用户提供了上下文时输出此模块）
> 这句话在原文中起什么作用？
> 说明以下一到两点（视情况选择最相关的）：
> - 它在文章结构中的位置：是论点？例证？转折？结论？情绪高潮？
> - 它和上下句的关系：承接什么？引出什么？回应什么？
> - 作者为什么在这里用这个句子——它的出现对读者理解有什么贡献？

**先感受整体**
用1句话说这句话"在做什么"——传递什么信息/情绪/意图。不是翻译，是功能描述。

**句子结构（谁在对谁做什么）**
用结构图或简洁标注拆解主干：
- 主语（谁/什么）：…
- 核心动作：…
- 作用对象/补充：…
- （如有从句）从句的作用：…

**关键表达 Spotlight**
挑出 1-2 个最值得注意的词/短语，单独解释它们在这个句子中的"贡献"。
不要解释每个词，只解释"如果你不懂这个，就会误解整句"的部分。

**语气与语境**
这句话的语气是什么（正式/口语/书面/情绪化）？
它可能出现在什么文章/对话类型中？

**变体理解**
用另一种英文表达传达类似意思（不是翻译成中文，而是英文改写），帮助用户感受"同一个意思的不同说法"。
```

## 通用规则（所有模式）

### 语言使用
- 解释语言：**中文**（因为用户是中文母语学习者，需要中文元认知辅助）
- 例句语言：**全英文**
- 绝对禁止：逐词对应式翻译（如"break → 打破，down → 下面"）

### 内容原则
1. **语义空间优先**：解释这个词/短语覆盖的"意义范围"，而不是单一定义
2. **场景驱动**：用真实语境而不是字典释义
3. **母语者视角**：描述母语者怎么"感受"这个词，而不是怎么"定义"它
4. **结构可视化**：句子解析要让用户看到"骨架"
5. **举一反三**：每次输出至少包含一个"延展"元素

### 输出长度
- 单词：~150-250 字
- 短语：~200-300 字  
- 句子：~250-400 字
- 保持紧凑，每个部分有实质内容，不要填充

## 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 用户输入的是中文问题（如"这个词什么意思"）但附带英文 | 识别英文部分，按对应模式处理 |
| 输入包含上下文（用户粘贴了整段文章） | 触发"在这篇文章里"模块，聚焦该词/句在上下文中的具体语义和作用 |
| 用户问的是语法问题而非词义 | 优先用句子模式的"结构"部分回答 |
| 非常基础的词（如 "the", "is"） | 解释其在当前语境中的具体用法，音标模块可省略 |
| 俚语/网络用语 | 重点说明使用场景和语气，标注 informal；音标标注口语发音变体 |
| 无上下文输入 | 跳过"在这篇文章里"模块，不要输出空的占位符 |
